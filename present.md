# 🎓 เอกสารประกอบการนำเสนอ (Project Presentation)
**โครงการ: ระบบวิเคราะห์การจราจรอัจฉริยะด้วย AI (Smart Traffic Analytics with Edge AI)**

---

## 1. 🌟 ทำไมต้องสร้างโปรเจกต์นี้? (Motivation & Purpose)
ในยุคปัจจุบัน ปัญหาการจราจรและความต้องการข้อมูลแบบ Real-time มีความสำคัญมาก การใช้มนุษย์มานั่งนับรถหรือดูกล้องวงจรปิดตลอด 24 ชั่วโมงนั้นไม่มีประสิทธิภาพและสิ้นเปลืองทรัพยากร
โปรเจกต์นี้ถูกสร้างขึ้นเพื่อ **"เปลี่ยนกล้องวงจรปิดธรรมดา ให้เป็นเซนเซอร์อัจฉริยะ (Smart Sensor)"** ที่สามารถ:
*   **นับจำนวนรถ** แยกประเภทได้อัตโนมัติ (รถยนต์, รถบัส, รถบรรทุก)
*   **วัดความเร็ว** ของการจราจรเพื่อดูสภาพคล่อง
*   **เก็บข้อมูลเชิงสถิติ** เพื่อนำไปวิเคราะห์วางแผนเมือง

### 🤝 ผู้ที่ได้รับประโยชน์ (Stakeholders)
1.  **เจ้าหน้าที่จราจร/ตำรวจ:** ตรวจจับรถกระทำผิด (เช่น ย้อนศร, จอดในที่ห้ามจอด) ได้ทันที
2.  **ห้างสรรพสินค้า/อาคาร:** วิเคราะห์จำนวนรถเข้า-ออก เพื่อบริหารจัดการลานจอดรถ
3.  **นักวางผังเมือง:** ใช้ข้อมูลสถิติ (Data) จากระบบไปคำนวณช่วงเวลาเร่งด่วน (Peak Hour) เพื่อปรับจังหวะสัญญาณไฟจราจร

## 2. ⚙️ หลักการทำงาน (System Architecture)
ระบบออกแบบมาในรูปแบบ **Edge AI** คือประมวลผลที่หน้างาน (On-premise) โดยไม่ต้องส่งภาพขึ้น Cloud ลดความหน่วง (Latency) และรักษาความปลอดภัยของข้อมูล

### 2.5 🛠️ เทคโนโลยีที่ใช้ (Technology Stack)
โปรเจกต์นี้เลือกใช้ Modern Stack ที่เป็นมาตรฐานอุตสาหกรรม และสามารถรันบน Edge Device ได้จริง:

*   **Programming Language:**
    *   **Python:** ภาษามาตรฐานสำหรับงาน AI/Data Science เขียนง่าย มี Library รองรับเยอะ
*   **Computer Vision & AI:**
    *   **OpenCV (cv2):** ใช้จัดการรูปภาพ วาดกราฟิก (Bounding Box) และรับ Input Video
    *   **YOLOv8 (Ultralytics):** Model AI ที่ใช้ในการตรวจจับรถ (Detection)
    *   **ByteTrack:** อัลกอริทึม Tracking ที่ใช้ติดตามวัตถุ
*   **User Interface (GUI):**
    *   **CustomTkinter:** สร้างหน้าจอแอปพลิเคชันที่สวยงาม (Modern UI/Dark Mode) แทน Tkinter เดิมๆ
*   **Data & Utilities:**
    *   **Pandas/CSV:** ใช้สำหรับการบันทึกและจัดการข้อมูลจราจร
    *   **NumPy:** ใช้คำนวณเวกเตอร์และคณิตศาสตร์สำหรับการนับรถ/ความเร็ว


**Flow การทำงาน:**
1.  **Input:** รับภาพวิดีโอจากไฟล์ (`.mp4`) หรือสตรีมสดจากกล้อง CCTV (`RTSP Protocol`)
2.  **Detection & Tracking:** ใช้ AI จับภาพรถและติดตามตำแหน่ง (ห้ามคลาดสายตา)
3.  **Business Logic:**
    *   **Counting Line:** ตรวจสอบว่าจุดศูนย์กลางของรถ (Centroid) ข้ามเส้นที่กำหนดหรือไม่
    *   **Speed Estimation:** คำนวณระยะทางพิกเซลเทียบกับเวลา เพื่อหาความเร็ว
4.  **Output:** แสดงผลบน Dashboard และบันทึก Log ลงไฟล์ Excel/CSV

## 3. 💻 โค้ดส่วนที่สำคัญ (Key Code Implementation)
หัวใจหลักของระบบอยู่ที่ไฟล์ `analyzer.py` ซึ่งทำหน้าที่เป็น "สมอง" ของ AI

```python
# ตัวอย่างโค้ดสำคัญ: การเรียกใช้ AI ในโหมด Tracking
results = self.model.track(
    frame, 
    persist=True,  # สำคัญมาก! บอกให้ AI จำรถคันเดิม (Tracking ID)
    classes=[2, 3, 5, 7], # กรองเฉพาะยานพาหนะ (Car, Bike, Bus, Truck) ตัดคนทิ้ง
    conf=0.5       # ความมั่นใจ > 50% ถึงจะนับ (กันเงา/แสงสะท้อน)
)
```

**ทำไมถึงสำคัญ?**
*   **`persist=True`**: ถ้าขาดตัวนี้ AI จะตาบอดทุกๆ เฟรม (Frame-by-frame) คือเห็นรถ A เป็นรถใหม่ตลอดเวลา ทำให้นับจำนวนผิดพลาดมหาศาล
*   **`conf=0.5`**: เป็นการจูน Hyperparameter เพื่อลด False Positives (การจับมั่ว) ทำให้ระบบแม่นยำขึ้นแบบก้าวกระโดด

## 4. 🧠 เจาะลึกเทคโนโลยี AI (AI Deep Dive)
ส่วนนี้คือเบื้องหลังความฉลาดของระบบ ประกอบด้วย 2 เทคโนโลยีหลัก:

### A. YOLOv8 (You Only Look Once - Version 8)
นี่คือโมเดล **Object Detection** ระดับ State-of-the-Art ที่ทำงานเร็วที่สุดในปัจจุบัน
*   **Architecture:** เป็นแบบ **CNN (Convolutional Neural Network)** ที่ถูก Train ด้วยภาพรถนับล้านรูป (COCO Dataset) ทำให้มัน "รู้จัก" ว่ารถหน้าตาเป็นอย่างไร โดยดู Pattern ของ ล้อ, ตัวถัง, กระจก
*   **Single-Stage Detector:** YOLO มองภาพทั้งภาพ "ครั้งเดียว" แล้วตอบได้เลยว่ามีรถกี่คัน อยู่ตรงไหนบ้าง (ต่างจากโมเดลยุคเก่าที่ต้องสแกนทีละจุด) ทำให้รันแบบ **Real-time** ได้ (30+ FPS)

### B. Object Tracking (BoT-SORT / ByteTrack)
แค่ YOLO อย่างเดียวไม่พอ เพราะ YOLO รู้แค่ว่า "ตรงนี้มีรถ" แต่ไม่รู้ว่า "คันนี้คือคันเดียวกับเมื่อกี้ไหม?"
เราจึงต้องใช้ **Tracker Algorithm** มาช่วย:
*   **Principle:** ทุกครั้งที่ YOLO เจอรถในเฟรมใหม่ Tracker จะคำนวณ:
    1.  **IoU (Intersection over Union):** กล่องสี่เหลี่ยมรอบรถซ้อนทับกับรถในเฟรมที่แล้วมากแค่ไหน?
    2.  **Kalman Filter:** ทำนายทิศทางการเคลื่อนที่ของรถล่วงหน้า ถ้ารถขยับไปตรงกับที่ทำนายไว้ -> ใช่คันเดิมแน่นอน!
    3.  **ReID (Optional):** ดูสีและรูปร่างว่าเหมือนเดิมไหม
*   **Result:** ทำให้ระบบสามารถแจก **Unique ID** (เช่น ID: 45) ให้รถคันนั้นติดตัวไปตลอด แม้จะขับผ่านเสาบัง หรือขับสวนกัน AI ก็ไม่สับสน
